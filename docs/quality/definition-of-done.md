# Definition of Done

A task is done when:

- [ ] It satisfies every accepted acceptance criterion.
- [ ] It stays within accepted scope or has accepted Change Requests.
- [ ] Relevant tests/checks pass or skipped checks are explained.
- [ ] Docs are updated if behavior changed.
- [ ] No unexpected dependencies were added.
- [ ] No secrets or generated junk were committed.
- [ ] PR template is complete.
- [ ] Human reviewer can understand the diff without reading the entire chat history.
